Issues to fix:

1. Customer confirmation button missing
When driver marks the order as delivered / arrived to customer, the driver screen shows:
"بانتظار تأكيد العميل"
But on the customer order details page, there is no visible button to confirm receipt.

Fix:
- On customer order details page, when deliveryStatus is:
  delivered_pending_confirmation
  or any existing equivalent waiting-customer-confirmation status,
  show a clear button:
  "تأكيد استلام الطلب"
- This button must call the fixed customer confirm receipt function.
- After confirmation, the order should no longer stay in active delivery state.

2. Complaint button must be locked after one complaint
Currently the complaint button can be pressed more than once for the same order.

Fix:
- Before showing/enabling "رفع بلاغ", check if a complaint already exists for this orderId in delivery_complaints.
- If a complaint exists:
  - Disable/hide the button.
  - Show a clear status text:
    "تم رفع بلاغ لهذا الطلب"
- Do not allow duplicate complaints for the same order from the same actor.
- Also do not show complaint button for completed orders.

3. Admin complaint details must be more Arabic and useful
The admin details modal currently shows raw English fields like:
ready_for_pickup
delivered_pending_confirmation
delivery_not_confirmed
driver

Fix:
- Display Arabic labels and Arabic translated values.
Examples:
  ready_for_pickup = جاهز للاستلام
  delivered_pending_confirmation = بانتظار تأكيد العميل
  delivery_not_confirmed = لم يتم تأكيد التوصيل
  driver = السائق
  customer = العميل
  pending/open = مفتوح
  resolved = تم الحل
  closed = مغلق

Admin needs a clear text box to write the solution/admin note.

Fix:

- Admin must be able to save it.
- When clicking "تعليم كمحلول", require a note if the note is empty.
- Save:

Notification text example:

Use the existing notifications structure if available. If not available, add a safe minimal notification under:
users/{uid}/notifications

7. Driver complaint flow must require a note
Currently the driver can raise a complaint without typing anything.

Fix:
- When driver taps "رفع بلاغ", open a modal/dialog using the same existing app dialog style.
- The dialog must include a required text input:
  "اكتب تفاصيل المشكلة"
- Do not submit if empty.
- Show validation:
  "يرجى كتابة تفاصيل المشكلة"
- Save the note into delivery_complaints.note
- source should be "driver"
- type can be "delivery_not_confirmed" or an existing compatible type.

8. Customer complaint flow must also require a note

- Open the same style modal.
- Require note.
- source should be "customer"
